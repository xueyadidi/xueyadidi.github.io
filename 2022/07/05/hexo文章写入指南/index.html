5 绑定域名

我们需要将网站内容绑定到自己的域名，需要如下几个操作：

5.0 将hexo中个人内容推送到github

5.0.1配置个人信息
git config --global user.name "XXXX"github注册时用户名
git config --global user.email "XXXXXXXXX@XXX.com"github注册时邮箱
5.0.2生成shh密钥,会在C盘/用户/这个文件夹下生成一个.ssh文件夹，该文件夹下会生成一个.pub的密钥文件
ssh-keygen -t rsa -C "XXXXXXXXX@XXX.com"
5.0.3查看ssh密钥内容
cat ~/.ssh/id_rsa.pub
5.0.4在github个人设置里添加对应的ssh keys
5.0.5修改hexo根目录下_config.yml文件中deploy部分，添加上之前的ssh：

5.1 将域名指向github.io域名

找到自己购买的域名，点击解析，构建记录类型CNAME，记录值为你的用户名.github.io的记录

5.2 github对应仓库中，修改解析域名

source文件中添加CNAME文件，重新添加解析域名。CNAME文件中为自己的域名。

个人遇到的问题：一开始采用的仓库中对应域名的方法是，github仓库中 settings-github_pages-custom_domian,填入自己的域名
重新hexo d 后 github 对应仓库中setting那一栏变成了空，然后发现github page项目上根目录下的CNAME文件被删了。
对应解决方案：
参考解决方案-csdn
逻辑：CNAME文件是github page 绑定域名需要添加的文件，在setting页面上设置会自动生成该文件。hexo会将public中的文件推送到远端仓库，并且完全覆盖该分支下的内容。也就说，在本地端public文件中没有CNAME文件，所以推送的时候远端上CNAME配置文件也会被删除，所以会导致404。
public中的文件是由source文件夹中的内容生成的，所以正确的绑定域名的方法应该在source文件下，新建CNAME文件。
